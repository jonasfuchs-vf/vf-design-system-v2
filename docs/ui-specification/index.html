#Naming conventions

#Files, folders, artboards and assets

No spaces or special characters are allowed to use in the filename. Use - instead of a space. Just use lowercase letters in filenames. American English is used for namings.

  • Markdown specification: [name].md, i.e. button.md
  • Sketch file: [name].sketch, i.e. button.sketch
  • Artboards in Sketch:
    • [name]-description
    • [name]-anatomy-[name-of-part]
    • [name]-variation-[name-of-variation]
    • [name]-state-[name-of-variation]
    • [name]-styling-[name-of-sub-category]-[name-of-style-description]
    • [name]-behavior-[name-of-behavior]
  • Assets:
    • [name]-description.png
    • [name]-anatomy-[name-of-part].png
    • [name]-variation-[name-of-variation].png
    • [name]-state-[name-of-variation].png
    • [name]-styling-[name-of-sub-category]-[name-of-style-description].png
    • [name]-behavior-[name-of-behavior].png

#Namings in specifications

Variations

[name]-size-[unit], i.e. promo-price-size-s

#File formats

  • Markdown (.md): Text-based file format to describe the UI element
  • Sketch (.sketch): Used to create assets/images for the specification for a better understanding of the UI
  • PNG (.png): Exported asset files from Sketch (size @1x). Will be linked in the Markdown file (how to).

#Folder structure

Design Token:

Folder: [design-token-name]

  • Sub-folder: assets
  • Sub-folder: sketch
  • Sub-folder: versions
  • File: design-token-name.md

Style:

Folder: [style-name]

  • Sub-folder: assets
  • Sub-folder: sketch
  • Sub-folder: versions
  • File: style-name.md

Component:

Folder: [component-name]

  • Sub-folder: assets
  • Sub-folder: sketch
  • Sub-folder: versions
    • File: component-name.md

Block:

  • Sub-folder: assets
  • Sub-folder: sketch
  • Sub-folder: versions
  • File: block-name.md

  • File: module-name.md
