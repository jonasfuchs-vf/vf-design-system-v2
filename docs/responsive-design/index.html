Responsive Design

#Why responsive Design?

Vodafone webpages behave responsive in all viewports below 1025px. This means that all elements and components are visible and usable on the user's device (mobile, phablet, tablet, desktop).

#Benefits

  • High consistency in user experience for Vodafone customers across all devices and screensizes at all times.
  • Responsiveness supports editorial work because only one source needs to be maintained for several screen views with the result of consistency and cost efficiency.
  • Responsive behaviour is a key requirement for search engine optimization.

The following principles define the responsive framework for all Vodafone webpages and provide guidance for everyone building webpages. These principles focus on our key-requirements: consistency, reusability, compatibility.

#Creating visual concepts

These viewport-widths are our recommended screensizes for building layouts with Sketch:

  • Mobile: 320px
  • Tablet: 768px
  • Desktop: 1280px

Important: During the design process for new components/elements it is essential to ensure always that also "critical" views are considered. Therefore the designer is advised to provide a DRAFT version the following viewports:

  • Mobile: approx. 580px
  • Tablet: approx. 620px

#Breakpoints

The Vodafone framework works with the following breakpoints:

MINIMUM supported viewport:

320px Mobile

320-599px----------------------------------------------------Breakpoint @600px

Tablet

600-1024px--------------------------------------------------Breakpoint @1024px

Desktop

1024px MAXIMUM supported viewport:

1025px

Larger viewports are filled with white space on the left and right sides with no further content-adaption. Benefits: Breakpoints are essential to provide an ideal user experience on all screens (mobile, phablet, tablet, desktop).
